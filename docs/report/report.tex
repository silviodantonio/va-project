\section{Introduction}
% <------------- README -------------> Press Alt+Z so that the text automatically adapts to your window size, don't use the Return button to break text lines in a static manner.
According to the Italian national news, road accidents are unfortunately becoming more and more frequent and they end up involving especially young drivers.

With this in mind, we aimed to develop an intuitive way to explore data on past accidents, with the goal of identifying patterns, which could support experts in making informed decisions in order to prevent new accidents. To this end, we developed a visual analytics system designed to address this purpose. The primary intended users of this solution are members of the Italian Ministry of Infrastructure and Transport. The project could be easily extended to support other countries as well, provided they have access to similarly structured national data.

The system provides both a high level overview and a detailed view of the several types and characteristics of accidents. This is accomplished by using a high level of interactivity, which enables users to do comparisons, extract details and easily search for patterns in the dataset.

The data used was provided by ISTAT and it's relative to the road accidents with injuries and deaths that occurred in Italy during 2024\footnote{\raggedright \istatDataset}.

\section{Dataset description and preliminary work}

\subsection{How the dataset is composed}

The dataset in use contains details about vehicle accidents with injuries
occurred in Italy during 2024. 
The attributes in the dataset are in summary: region and time (hour, weekday,
month) of accident, road section where the accident occurred (traffic circle,
intersection, etc...), type of accident (between vehicles,
vehicle-pedestrian, etc...), deadliness of the accident, and number of
accidents registered with the given characteristics.

\subsection{How data was cleaned and manipulated}

To effectively explore and interpret this dataset, we first performed a data cleaning and optimization process. A key challenge was the dataset's attributes; to address this, we implemented a dimensionality reduction strategy.
However, the fact that most of these attributes are categorical posed a
challenge when we attempted to do so.
In particular, instead of working with verbose categorical strings, we pruned the tuples to retain only integer-based indices representing specific attributes.
By operating directly on these encoded indices, we significantly minimized the computational overhead. This choice ensured higher performance and responsiveness during the interactive exploration of the dataset.
In particular, the dataset cleaned and filtered from the data
published on the ISTAT website is comprises of 92.000 tuples of 8
attributes each.

\subsection{Categorical Analysis through Interactive Bar Charts}
Bar Charts serve as the primary tool for representing categorical distributions, specifically for \textbf{Mortality Analysis}, \textbf{Regional accidents}, \textbf{Road Intersections}, and \textbf{Accident Types}. This choice leverages the human visual system's proficiency in \textit{magnitude comparison} along a common baseline, facilitating the immediate identification of the most frequent accident scenarios.
These charts implement a \textbf{dynamic dual encoding}:
\begin{itemize}
    \item \textbf{Visual Layering and Feedback:} Each bar displays the absolute number of accidents. Upon interaction, the system overlays a secondary bar representing the filtered subset, providing a direct visual comparison with the original distribution. Concurrently, textual labels update to show the precise percentage of the selection relative to the specific category.
    \item \textbf{Trend Indicators:} To provide statistical context, we introduced symbolic indicators ($\blacktriangle, \blacktriangledown, \approx$). These compare the filtered subset's distribution against the national baseline, signaling whether a specific category is over-represented, under-represented, or stable.
\end{itemize}

\subsection{Geospatial Mapping and Redundant Encoding}
The distribution of accidents across Italian regions is represented through a \textbf{Choropleth Map} complemented by the \textbf{Regional Bar Chart}. This intentional \textit{redundant encoding} is designed to balance numerical accuracy with spatial intuition:
\begin{itemize}
    \item \textbf{Ranking and Precision:} The Bar Chart provides a quantitative ranking, allowing for a more precise comparison between regions and the identification of numerical variances that color intensity alone might obscure.
    \item \textbf{Geospatial Context:} The Choropleth Map enables the identification of macro-regional clusters and spatial correlations, providing essential geographical context that a simple ranking fails to highlight.
\end{itemize}
Through interaction with a region in the map, the region's name and exact incident count will be displayed, while clicking a region propagates a real-time filter across the entire dashboard.

\subsection{Temporal Mapping and Cross-Filtering}
Temporal patterns are explored through two \textbf{Heatmaps}, which leverage color encoding to make seasonal trends and peak hours immediately visible. To facilitate the analysis, the heatmaps support a \textbf{multi-level selection logic}:

\begin{itemize}
    \item \textbf{Flexible Selection:} Users can interact with the matrix by selecting individual cells, entire rows, or full columns. This allows for the isolation of both specific temporal coordinates.
    
    \item \textbf{Global Synchronization:} Any temporal selection propagates a real-time filter across the entire dashboard.
    
    \item \textbf{Dual encoding:} Each cell provide the exact incident counts and temporal coordinates, ensuring that the visual density of the heatmap is always backed by numerical data.
\end{itemize}

\subsection{Dimensionality Reduction and Brushing: PCA}
To address the high dimensionality of the dataset, we employed \textbf{Principal Component Analysis (PCA)} to project multidimensional observations—incorporating mortality, accident types, and road characteristics—into a continuous 2D space. 
Given the categorical nature of most attributes and the significant volume of the dataset, we implemented specific strategies to maintain visual clarity:
\begin{itemize}
    \item \textbf{Color Encoding and Overplotting:} To manage the presence of identical points we utilized \textbf{color intensity} as a quantitative measure. This approach effectively visualizes data density, revealing clusters that would otherwise be hidden by \textit{overplotting}.
    
    \item \textbf{Performance Optimization:} To ensure fluid, real-time interaction, the PCA visualization is rendered using a \textbf{Canvas-based approach} rather than SVG. This allows the system to handle thousands of data points without compromising the responsiveness of the dashboard.
\end{itemize}

Interaction with this latent space is managed through \textbf{Brushing}:
\begin{itemize}
    \item \textbf{Cluster Isolation:} By selecting a sub-region of the PCA canvas, the user can analyze specific observations sharing similar underlying characteristics.
    
    \item \textbf{Global Recalculation:} This interaction triggers a global update; all numerical values, percentages, and trend symbols in the other components are instantly recalculated to reflect the specific attributes of the brushed observations.
\end{itemize}

\subsection{Global Selection Analytics}
To maintain context during multi-modal exploration, a \textbf{global status indicator} provides an immediate overview of the active filters. It displays the selection in the format: \textit{Selected: X\% (N / Total)}, where both the percentage and the fraction represent the proportion of incidents currently analyzed relative to the entire dataset.
